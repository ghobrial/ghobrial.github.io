🛡️ How to Prevent This Attack

URI Validation Requirements:

  • Scheme: Always allow secure schemes (e.g. https://). Scrutinize other protocol usage. Other protocols include javascript://, content://, file://.
  • Host: Validate against whitelist of trusted domains. Be careful of allowing arbitrary subdomains.
  • Path: Sanitize and validate all path components.
  • Parameters: User-provided code or scripts should never be allowed to run.

Ensure all portions of the URI are validated and not just certain portions.

Result: Credential theft via phishing and code execution are prevented!
